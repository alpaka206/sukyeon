# Handoff: 석연MRO (Sukyeon MRO) 기업 웹사이트

## Overview
알루미늄 다이캐스팅 이형제 전문 제조사 **석연MRO**의 기업 웹사이트 풀 리뉴얼 디자인입니다. 기존 10년 된 사이트를 현대적인 B2B 제조사 사이트로 재설계했습니다. 회사소개, 제품 카탈로그, 제조공정, 자료실(MSDS), 견적 문의 등 11개 페이지를 단일 클라이언트사이드 라우팅 SPA로 구성했습니다.

- 톤: 깔끔하고 전문적(신뢰감 중심)
- 컬러: 딥 네이비 + 시안(teal) 포인트
- 언어: 한국어
- 타깃: 다이캐스팅 공장·주조 현장 등 B2B 고객

## About the Design Files
이 번들의 파일은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 입니다. 의도한 외형·동작·레이아웃·카피를 보여주는 참고용이며, 그대로 프로덕션에 복사해 쓰는 코드가 아닙니다.

작업 목표는 이 HTML 디자인을 **타깃 코드베이스의 기존 환경(React/Vue/Next.js 등)에 그 환경의 관행대로 재구현**하는 것입니다. 아직 코드베이스가 없다면, 기업 사이트에 적합한 프레임워크(예: Next.js + React)를 선택해 구현하면 됩니다. 정적/SSG 사이트로도 충분합니다.

원본은 단일 파일 `수경MRO 웹사이트.dc.html` 안에 자체 템플릿 런타임(`support.js`)으로 작성돼 있습니다. 이 런타임 자체를 이식할 필요는 없고, **마크업·인라인 스타일·상호작용 로직을 타깃 프레임워크의 컴포넌트로 옮기면** 됩니다.

## Fidelity
**High-fidelity (hifi).** 최종 컬러·타이포그래피·간격·상호작용이 모두 확정된 픽셀 단위 목업입니다. 개발자는 코드베이스의 기존 라이브러리/패턴을 사용하되 이 디자인을 픽셀에 가깝게 재현해야 합니다. 단, 일부 이미지(공장/대표 사진, 고객사 로고, 인증서, 지도, 카탈로그 PDF, MSDS PDF)는 **플레이스홀더**이며 실제 자산으로 교체 필요(아래 Assets 참조).

## Interactions & Behavior
- **클라이언트사이드 라우팅**: 상단 메뉴/푸터/CTA 클릭 → 현재 페이지 전환(state.page). 페이지 전환 시 `window.scrollTo(0,0)`.
  - 타깃 프레임워크에서는 실제 라우팅(Next.js App Router 등)으로 구현 권장. 각 페이지는 `/`, `/about`, `/business`, `/process`, `/products`, `/data`, `/clients`, `/cert`, `/news`, `/catalog`, `/contact` 정도로 매핑.
- **드롭다운**(사업분야/고객지원): `onMouseEnter`로 열고 `onMouseLeave`로 닫음. 패널은 흰 카드 + shadow `0 18px 44px -14px rgba(10,27,51,0.28)`. 항목 hover 시 bg `#f1f5f9`/text `#0fb0c8`.
- **좌측 LNB(회사소개/제품안내)**: 항목 클릭 → 해당 섹션 id로 smooth scroll(헤더 높이 보정 ~92px). **스크롤 스파이**: 스크롤 시 각 섹션의 `getBoundingClientRect().top <= 150`인 마지막 섹션을 활성으로 표시.
- **히어로 캐러셀**: `setInterval` 4500ms로 슬라이드 +1(홈에서만 작동). 점 클릭 시 해당 인덱스. 페이드 전환 `opacity 0.9s ease`.
- **버튼/링크 hover**: 텍스트색 `#0a1b33`로 진해짐, 아웃라인 버튼은 보더/텍스트 `#0fb0c8`.
- **반응형**: 현재 데스크톱 고정폭(≈1280+) 기준. 모바일은 미구현 — 햄버거 메뉴, LNB 상단 가로 탭/접힘, 히어로 1열, 그리드 1열 등 별도 작업 필요.

## State Management
원본의 단일 컴포넌트 상태(타깃에서는 라우팅 + 페이지별 로컬 상태로 분해 권장):
- `page`: 현재 페이지 키(`home`/`about`/`business`/`process`/`products`/`msds`/`clients`/`cert`/`news`/`location`(→about 섹션)/`contact`/`catalog`).
- `openMenu`: 열린 드롭다운(`about`(미사용)/`business`/`products`(미사용)/`support`/null).
- `aboutSection`: 회사소개 활성 섹션(`greeting`/`info`/`history`/`location`).
- `productSection`: 제품안내 활성 섹션(`release`/`pranza`/`oils`/`parts`).
- `slide`: 히어로 캐러셀 현재 인덱스(0–3).
- 폼은 미연동(목업). 실제 구현 시 견적 폼 → 이메일/백엔드 전송, 유효성 검사(필수: 회사명/담당자/연락처/내용 + 개인정보 동의) 필요.

## 구현 권장
- 프레임워크: Next.js(App Router) + React + TypeScript, 정적 배포(SSG).
- 스타일: 인라인 값 그대로 토큰화(Tailwind config 또는 CSS 변수). 색/타이포/간격은 위 토큰 사용.
- 폰트: Pretendard + IBM Plex Mono를 `next/font` 또는 CDN으로.
- 라우팅: 페이지별 실제 라우트. 회사소개/제품안내는 단일 라우트 + 섹션 앵커 + 스크롤스파이(IntersectionObserver 권장).
- 접근성: 드롭다운/캐러셀 키보드·포커스 처리, 폼 라벨/에러, 이미지 alt 보완.
- 반드시 추가 작업: **모바일 반응형**(현재 데스크톱 전용).
